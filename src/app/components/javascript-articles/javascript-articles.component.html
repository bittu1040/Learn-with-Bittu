6. Explain the difference between == and === operators in JavaScript.

The == operator checks for equality, but it performs type coercion if the operands are of different types. On the other hand, the === operator checks for equality without performing type coercion, meaning both value and type must be the same for it to return true.

12. Promise Methods ?

Promise methods are fundamental to asynchronous JavaScript programming, providing a way to handle asynchronous operations and their results.

Promise.resolve(): This method returns a Promise object that is resolved with a given value. It is commonly used to convert non-Promise values into Promise objects, allowing consistent handling of asynchronous operations.

Promise.reject(): Conversely, Promise.reject() returns a Promise object that is rejected with a given reason (typically an Error object). It is useful for signaling that an asynchronous operation has failed or encountered an error condition.

Promise.all(): Promise.all() takes an iterable of Promises as input and returns a single Promise that resolves when all of the input Promises have resolved, or rejects if any of the input Promises reject. It is commonly used when you need to perform multiple asynchronous operations concurrently and wait for all of them to complete before proceeding.

Promise.race(): In contrast, Promise.race() takes an iterable of Promises and returns a Promise that resolves or rejects as soon as one of the input Promises resolves or rejects, with the value or reason from that Promise. It is useful for scenarios where you want to respond as soon as any one of multiple asynchronous operations completes.

Understanding and effectively using these Promise methods is crucial for writing robust and efficient asynchronous JavaScript code, enabling developers to handle asynchronous operations gracefully.

18. async-and-defer ?

The attributes `async` and `defer` are used in HTML script tags to control the loading and execution of external JavaScript files.

async: When the `async` attribute is present, the script will be fetched asynchronously, meaning it will not block the parsing of the HTML document. The script will be executed as soon as it is downloaded, regardless of the order it appears in the document.

defer: The `defer` attribute, similar to `async`, also allows the script to be fetched asynchronously. However, scripts with the `defer` attribute will only be executed after the HTML document has been fully parsed. Scripts with `defer` maintain their order of execution as they appear in the document.

Using `async` and `defer` attributes can help improve page loading performance by allowing scripts to be loaded in parallel with other resources without blocking rendering or delaying the document's interactive state.

20. Debounce and Throttle ?

The techniques of 'debounce' and 'throttle' are used in web development to control the frequency of execution of a function, typically used in event handling scenarios.

Debounce: When using debounce, the execution of a function is delayed until after a certain amount of time has passed since the last invocation of the function. This is particularly useful when dealing with events such as user input, where you want to wait for a pause before triggering an action to avoid excessive calls, such as search suggestions in a search bar.

Throttle: Throttle, on the other hand, limits the rate at which a function can be called. It ensures that the function is executed at most once per specified time interval. This is beneficial for scenarios like scroll events, where you want to ensure that a function is not called too frequently, preventing performance issues.

Implementing debounce and throttle techniques can optimize performance and improve the user experience by controlling the frequency of function execution, especially in situations with high event triggers.

23. Deep Copy and Shallow Copy in JavaScript

Deep copy and shallow copy are essential concepts in JavaScript, particularly when dealing with nested data structures like arrays of objects or objects containing other objects.

Shallow Copy

A shallow copy creates a new object but only copies references to the nested structures. This means that modifications made to the nested structures in the copied object will also affect the original object, and vice versa.

Deep Copy

A deep copy, on the other hand, creates a new object with its own values, completely independent of the original object, including all nested structures. Changes made to the copied object will not affect the original object, and vice versa.
